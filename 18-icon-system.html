Icon System

The complete icon system for Koleex. Every icon must communicate clarity, precision, and professionalism. No decorative styles, no artistic effects — only clean, geometric, system-oriented iconography.

Icon System

Systematic Iconography for an Industrial Brand

Koleex icons are not decorative elements — they are functional UI components. Every icon must be clean, geometric, consistently weighted, and purpose-driven. Icons across all categories follow strict construction rules to ensure visual harmony and usability.

Icon Style Philosophy

Koleex icons must feel engineered, not designed. They serve a functional purpose within user interfaces and brand materials. Every icon should communicate with clarity and precision, never with personality or decoration.

Icon Construction Rules

Every Koleex icon is built using strict geometric principles. Construction rules ensure visual consistency across all icon categories and sizes. Icons that violate these rules will be rejected during brand review.

Built Using

Grid system — all icons built on a consistent pixel grid
Geometric shapes — circles, squares, rectangles as base forms
Straight lines — clean horizontal and vertical strokes
Simple curves — minimal, predictable arc geometry
Consistent stroke — uniform 2px weight across all icons
Consistent corner radius — unified rounding on all corners
Consistent proportions — same visual weight at every size
Optical alignment — visually centered, not mathematically

Must NOT Include

Too many details — keep icons simple and readable at small sizes
Textures — no dot patterns, hatching, or surface effects
Shading — no light/dark gradation within icon shapes
Gradients — no color transitions or multi-tone fills
Shadows — no drop shadows, inner shadows, or glow effects
Perspective — no 3D or isometric icon views
Complex illustration — no detailed pictorial representations
Text inside icons — no letters or numbers embedded in icons
Inconsistent line thickness — all strokes must match exactly
Icon Grid Construction

Every icon is constructed within a standardized grid system. The grid ensures consistent sizing, padding, and optical alignment across all icons in the system. Icons must respect the padding zone and never bleed into the grid edges.

Stroke Style

Stroke weight is the most critical element of icon consistency. All Koleex icons use a 2px stroke weight at the standard 24px canvas size. Deviations in stroke weight break visual harmony and are not permitted.

Icon Alignment with Text

Icons must be optically aligned with adjacent text. The standard gap between an icon and its label is 12px. Tighter or wider spacing breaks the visual rhythm and must be avoided.

Icon Do

These are the 20 rules that must be followed for every Koleex icon. These rules ensure consistency, clarity, and professional quality across all icon usage.

Use consistent 2px stroke weight across all icons at standard size
Build all icons on the standardized pixel grid system
Use geometric shapes as the foundation for all icon construction
Maintain consistent corner radius across all icons in the set
Optically center icons within their bounding box, not mathematically
Use monochrome icons — single color per icon at all times
Respect the padding zone — never let icon art touch grid edges
Use only the five predefined sizes: 16px, 20px, 24px, 32px, 40px
Align icons with text using 12px standard spacing gap
Use dark icons on light backgrounds, white icons on dark backgrounds
Reserve accent colors only for interactive states (active, error, etc.)
Keep icons simple — recognizable at the smallest size (16px)
Test every icon at all five sizes before final approval
Use icons from the approved Koleex icon library only
Pair icons with text labels in navigation and primary actions
Maintain visual weight consistency across all icon categories
Export icons in SVG format for all digital applications
Use pixel-perfect alignment for icons at every size
Follow the icon naming convention for all files and components
Get brand team approval before adding any new icon to the system
Icon Don't

These are the 20 mistakes that must never appear in any Koleex icon usage. Any icon violating these rules will be rejected during the brand review process.

Don't use gradients, shadows, or 3D effects on any icon
Don't use multi-colored icons — all icons must be monochrome
Don't use decorative, playful, or cartoon-style icon sets
Don't use hand-drawn or sketch-style icons in any material
Don't use emoji as substitutes for system icons
Don't mix icon styles from different icon libraries in one interface
Don't use skeuomorphic or realistic icon styles
Don't scale icons to non-standard sizes outside the five predefined
Don't use inconsistent stroke weights across icons in the same view
Don't place icons too close to text — always maintain 12px minimum gap
Don't rotate icons unless rotation is part of the icon's meaning
Don't add colored backgrounds or circles behind icons arbitrarily
Don't use text or numbers embedded inside icon shapes
Don't use overly detailed or complex illustrations as icons
Don't use glossy, shiny, or reflective effects on icons
Don't use icons without clear functional purpose — no decoration
Don't stretch or distort icon proportions for any reason
Don't use heavy drop shadows or outer glow effects on icons
Don't create custom icons without following the grid construction rules
Don't use icons from unapproved or unlicensed icon sources
