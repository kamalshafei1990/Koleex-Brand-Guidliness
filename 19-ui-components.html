UI Components & Interface Style

The complete UI component system for Koleex. Every interface element must communicate clarity, precision, and professionalism. No decorative styles, no artistic effects — only clean, structured, system-oriented components.

UI Components

Systematic Interface Design for an Industrial Brand

Koleex UI components are not decorative elements — they are functional building blocks. Every component must be clean, structured, consistently styled, and purpose-driven. Components across all categories follow strict design rules to ensure visual harmony, usability, and enterprise-grade professionalism.

Interface Design Philosophy

The Koleex interface philosophy is rooted in engineering precision. Every UI decision must serve a functional purpose. The interface exists to help users complete tasks efficiently — not to impress with visual artistry. Form follows function at every level.

Must Prioritize
  • Readability — text must be legible at all sizes
  • Usability — every element must be easy to interact with
  • Structure — clear visual hierarchy on every page
  • Hierarchy — important content appears first
  • Consistency — same patterns across all screens
  • Alignment — all elements snap to the grid
  • Spacing — consistent padding and margins
  • Grid layout — modular, predictable structure
  • Modular components — reusable building blocks
  • Predictable behavior — no UI surprises
  • Clear navigation — users always know where they are
  • Efficiency — minimize clicks to complete tasks
  • Data clarity — numbers and data are easy to read
  • Minimal distractions — no unnecessary elements
  • Professional appearance — enterprise-grade look
Should NOT Prioritize
  • Decoration — no ornamental elements
  • Colorful design — restrained palette only
  • Visual effects — no unnecessary flair
  • Animations everywhere — motion must be purposeful
  • Trendy UI — no following design fads
  • Heavy gradients — flat, clean surfaces only
  • Glass effects — no glassmorphism
  • Artistic layouts — function over form
  • Unusual navigation — conventional patterns only
  • Experimental UI — proven patterns preferred
UI Layout Structure

The standard Koleex UI layout follows a fixed structure: left sidebar for navigation, top bar for context and actions, page header for the current section title, main content area for data and forms, and a footer for secondary information. All layouts must conform to this structure.

Sidebar Navigation

The sidebar is the primary navigation component in all Koleex dashboard interfaces. It is fixed on the left, dark-themed, and contains the brand logo, menu items with icons, and a user area at the bottom. Width is 220px. Each menu item is 44px tall with 16px horizontal padding.

Buttons are the primary interactive elements in Koleex interfaces. Each button type serves a specific purpose. Form System

Forms are critical data-entry interfaces in Koleex. All form elements must be clean, accessible, and consistently styled. Labels appear above inputs with 8px gap. Inputs use 1px gray borders, 6px radius, and 40px height. Focus states use a black border with a subtle shadow ring. Consistent 20px spacing between form groups.

Table System

Tables are the primary data display component in Koleex dashboards. They must be clean, scannable, and information-dense without being cluttered. Headers use uppercase gray text. Rows alternate backgrounds for readability. Status badges use color-coded pills. Amounts are right-aligned with tabular numerals.

UI Do

These are the 23 rules that must be followed for every Koleex interface. These rules ensure consistency, usability, and professional quality across all UI components and screens.

Use consistent 8px border radius on all cards, inputs, and buttons
Maintain 24px internal padding on all card components
Use the 4px spacing scale for all margins and padding values
Keep all interactive elements at minimum 40px touch target height
Use black (#000000) as the primary action color for buttons
Place labels above form inputs with 8px gap consistently
Use status badges with semantic colors — green for success, red for error
Right-align all monetary values and numbers in tables
Use alternating row backgrounds in data tables for readability
Include pagination for any table with more than 10 rows
Show clear focus states on all interactive elements for accessibility
Use the sidebar navigation pattern for all dashboard interfaces
Keep the top bar height at exactly 60px across all screens
Use Inter as the only font family in all UI components
Follow the typography hierarchy — page title, section title, card title, body, label, caption
Use 1px borders (#E8E8ED) for all card and input outlines
Include search and filter functionality on all data tables
Use modals for confirmations and destructive actions with clear cancel option
Show loading states for all asynchronous operations
Use toast notifications for success/error feedback — positioned top-right
Ensure all forms have proper validation with inline error messages
Maintain consistent hover states on all clickable elements
Use breadcrumbs for navigation depth beyond 2 levels
UI Don't

These are the 22 mistakes that must never appear in any Koleex interface. Any UI violating these rules will be rejected during the design review process.

Don't use gradients on buttons, cards, or any UI surface
Don't use glassmorphism, neumorphism, or other trend-based effects
Don't use drop shadows heavier than the predefined shadow scale
Don't use colored backgrounds on cards — white or gray-50 only
Don't use more than 3 font weights on a single screen
Don't use animation durations longer than 350ms for UI transitions
Don't use custom colors outside the defined UI color palette
Don't mix different icon styles in the same interface
Don't use rounded corners larger than 12px on any component
Don't place buttons inside table cells — use row actions or toolbar
Don't use inline editing without a clear save/cancel mechanism
Don't use horizontal navigation tabs as the primary navigation
Don't use auto-playing carousels or sliders in dashboard interfaces
Don't use decorative illustrations or mascots in the UI
Don't use floating action buttons (FABs) — use toolbar buttons instead
Don't use collapsible sidebar as default — sidebar must always be visible on desktop
Don't use placeholder text as the only label for form inputs
Don't use more than 2 primary buttons on the same screen
Don't use alert dialogs for non-critical information — use toast instead
Don't use pagination without showing total count and current range
Don't use custom scrollbars that differ from the system default style
Don't use font sizes smaller than 11px anywhere in the interface
