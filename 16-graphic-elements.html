Graphic Elements System

The complete system of graphic elements, visual components, and structural patterns that define the Koleex visual language. Every line, frame, card, icon, and pattern follows strict rules.

Visual System

Engineered Graphics for an Industrial Brand

Koleex graphic elements are not decorative — they are structural. Every visual component serves a purpose: to organize, separate, highlight, or frame content. The graphic language is minimal, geometric, and precise.

Graphic System Philosophy

The Koleex graphic system is built on engineering principles, not artistic expression. Every element must feel precise, controlled, and purposeful. The visual language draws from industrial design, technical documentation, and modern UI systems.

Line System

Lines are the core identity element in the Koleex graphic system. They create structure, divide content, frame elements, and guide the eye. All lines must be thin, precise, and functional.

Lines must always be straight — no curves, waves, or organic shapes
Line weight: 0.5px to 2px maximum (never thicker)
Lines are always horizontal or vertical — no diagonal lines except in patterns
Line color: Black, gray shades, or white on dark backgrounds only
Lines serve a purpose: dividing, framing, highlighting, or aligning
Never use lines as pure decoration without structural function

Frame System

Frames contain, organize, and highlight content. Koleex uses four frame types, each with a specific role. Frames must always be rectangular with straight edges and consistent border widths.

Frames must be rectangular — no circles, ovals, or irregular shapes
Border radius: 0px to 12px maximum
Border width: 1px to 2px — never thicker
Frame colors: black, gray, or white only
Frames must have consistent padding inside (minimum 16px)
Never use double borders, bevels, or 3D effects on frames

Card System

Cards are the most important graphic component in the Koleex design system. They are used everywhere: websites, documents, presentations, dashboards, and UI. Cards organize, contain, and present information in structured, scannable units.

Cards for product listings and feature descriptions
Cards for team members, testimonials, and profiles
Cards for KPI data, statistics, and metrics
Cards for comparison tables and feature matrices
Cards for navigation items, menus, and CTAs
Cards for blog posts, articles, and content previews
Card Rules: Background white or gray-50 only. Border 1px solid gray-200. Border-radius 8-12px. Padding minimum 16px. No colored backgrounds. No heavy shadows. No gradients. No rounded corners beyond 12px.

Divider System

Dividers separate content sections and create visual rhythm. They must be minimal and consistent. Never use dividers as decoration — only as structural separators.

Dividers are always horizontal lines — full width or partial width
Color: gray-200 to gray-300 only (never black for full-width dividers)
Weight: 0.5px to 2px maximum
Always include vertical spacing above and below dividers

Divider Types

Standard Divider (1px solid)
Section Divider (1px + spacing)
Subtle Divider (0.5px light)
Accent Divider (2px short bar)
Icon System Style

Icons in the Koleex system must be consistent, monochrome, and geometric. They follow a strict style guide to maintain visual unity across all touchpoints.

Icon style: line-style (outline) preferred, solid acceptable
Icon color: monochrome black or white only — never multicolor
Icon size: consistent within context (16px, 20px, 24px, 32px)
Icon weight: consistent stroke width across all icons in a set
Icon grid: all icons must align to a consistent grid
Never use colorful, cartoon, or 3D icons

Background Graphic Elements

Background elements add depth and visual interest without competing with foreground content. They must be subtle, monochrome, and geometric. Backgrounds should enhance, not distract.

Allowed Background Elements
  • Subtle dot grid patterns at low opacity
  • Light line grid overlays
  • Geometric shapes at very low opacity (5-10%)
  • Gradient overlays (black to transparent only)
  • Blueprint-style technical grids
  • Subtle radial gradients for depth
  • Monochrome photography with overlay
  • Solid color backgrounds (black, white, gray)
Not Allowed Background Elements
  • Colorful gradients or rainbow effects
  • Busy textures (wood, marble, fabric)
  • Stock photos without proper overlay
  • Animated or moving backgrounds
  • Pattern fills that compete with text
  • Watermarks or repeated logos as pattern
  • Organic or nature-inspired textures
  • High-contrast background patterns
Highlight Elements

Highlights draw attention to important content, key data, or calls to action. They must be used sparingly and consistently. Over-highlighting defeats the purpose.

Underline bars for section titles and key headings
Side highlights (left border accent) for callouts and quotes
Corner brackets for featured images or diagrams
Highlight boxes (light gray background) for important text
Tag labels (small pill badges) for categories and status
Progress lines for completion status and data visualization
Highlight colors: Black or gray only. Never use colored highlights except for success (green) and error (red) states in UI contexts.

Diagram Style

Diagrams communicate processes, systems, and relationships. They must be clean, monochrome, and structured. All diagram elements follow the same graphic rules as the rest of the brand system.

Diagrams must use rectangular boxes with consistent styling
Arrows and connectors must be straight lines (no curved arrows)
Color: black, white, and gray only — no colorful nodes
Text inside boxes must use Inter font, consistent size
Flow direction: left to right or top to bottom
Never use 3D effects, shadows, or gradients in diagrams

Graphic Elements Do

These are the 22 rules that must be followed in every Koleex design when using graphic elements. These rules ensure consistency, clarity, and brand integrity across all touchpoints.

Use straight, geometric lines only — never curved or wavy
Keep all graphic elements monochrome (black, white, gray)
Use consistent border widths within the same design (1px or 2px)
Maintain consistent padding inside all frames and cards (minimum 16px)
Use grid-based patterns only (dots, lines, diagonals)
Keep icons simple, geometric, and consistent in weight
Use subtle backgrounds that don't compete with content
Align all graphic elements to the grid system
Use dividers to create visual rhythm and separation
Apply highlights sparingly for maximum impact
Use corner brackets and accent bars for emphasis
Keep diagrams clean, minimal, and left-to-right or top-to-bottom
Use cards as the primary content container for all UI and documents
Maintain consistent border-radius across all elements (8-12px)
Use shadows only when needed for elevation (subtle, never heavy)
Keep all visual elements purposeful — every element must have a function
Use the official icon set for all interface and document icons
Maintain visual hierarchy through size, weight, and spacing
Test all graphic elements on both light and dark backgrounds
Use consistent spacing between graphic elements (follow the spacing system)
Document all custom graphic elements in the brand asset library
Get brand team approval for any new graphic element before use
Graphic Elements Don't

These are the 22 mistakes that must never appear in any Koleex design. Violating these rules will result in rejected designs during brand review.

Don't use curved, wavy, or organic lines
Don't use colorful graphic elements (no red, blue, green, etc.)
Don't use heavy borders or thick outlines (never more than 2px)
Don't use drop shadows, glows, or 3D effects on graphic elements
Don't use gradients on borders, lines, or frames
Don't use decorative patterns (floral, paisley, tribal, etc.)
Don't use cartoon, hand-drawn, or illustrative icons
Don't mix icon styles within the same design
Don't use rounded corners beyond 12px (no pill shapes for containers)
Don't use colored backgrounds on cards (white or gray-50 only)
Don't overcrowd layouts with too many graphic elements
Don't use graphic elements purely for decoration
Don't use inconsistent spacing between graphic elements
Don't use diagonal lines in layouts (only allowed in patterns)
Don't use emoji or pictographic icons in professional materials
Don't use noisy or high-contrast background patterns
Don't use animated graphic elements in static documents
Don't use multiple highlight styles on the same page
Don't place graphic elements off-grid
Don't use bevels, embossing, or texture effects
Don't copy graphic styles from other brands
Don't publish any design with unapproved graphic elements
