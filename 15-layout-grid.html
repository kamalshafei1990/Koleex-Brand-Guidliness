Layout & Grid System

The official layout and grid system defining how all content is structured, spaced, and aligned across every Koleex brand touchpoint — from posters and websites to software interfaces, documents, and presentations.

LAYOUT & GRID IDENTITY SYSTEM

Structure before decoration. Every element must have a reason to be where it is.

The Koleex layout system is the structural backbone of the entire brand identity. Every piece of content — from a poster to a software dashboard — must follow a precise, grid-based layout that communicates professionalism, engineering discipline, and visual clarity. Layout is not optional. It is the first thing that defines whether a design looks like Koleex or not.

Grid-Based Precision Structured Breathing Space Modular Consistent Spacing Responsive Premium Industrial
Layout Philosophy

Structure before decoration. The Koleex layout philosophy is rooted in engineering precision. Every element must have a clear reason for its position, size, and spacing. Layout defines the visual order of the brand before any color, typography, or imagery is applied.

This means:

Every element is placed on a grid — no freehand positioning
Spacing is consistent and based on a defined scale
Margins and padding follow strict rules
Content blocks are modular and repeatable
Alignment is always left or center — never random
White space is intentional and generous
Hierarchy is established through size and position, not decoration
The layout should feel engineered, not artistic
Every layout decision must serve clarity and readability
The system must scale across all media and screen sizes

The layout should feel:

Composed Controlled Balanced Rational Clean High Quality Scalable System-Driven
Core Layout Principles

These 20 principles govern every layout decision across the Koleex brand. They must be followed without exception in all design work — digital, print, environmental, and documentation.

Use a 12-column grid as the foundation for all layouts
Maintain consistent margins on all sides of every layout
Use a defined spacing scale — never arbitrary values
Align all elements to the grid — no freehand placement
Prioritize left alignment for body content and descriptions
Use center alignment only for hero sections and headlines
Ensure generous white space between sections
Create clear visual hierarchy through size and position
Use modular content blocks that can be reused
Keep content width controlled — never edge-to-edge text
Establish rhythm through consistent vertical spacing
Group related elements together with proper proximity
Separate unrelated elements with clear spacing or dividers
Design for responsiveness — layouts must adapt to all screen sizes
Use consistent gutters between columns
Never crowd content — if in doubt, add more space
Treat negative space as a design element, not empty space
Follow the section structure pattern for all documentation
Ensure layouts look like Apple, BMW, Sony, IBM level quality
Every layout must pass the "engineering precision" test

Content Width & Margins

Content width and margins are critical to the Koleex brand. Content must never stretch edge-to-edge. Generous margins create breathing space and give every layout a premium, professional feel.

Maximum content width: 1200px for web and digital layouts
Minimum side margins: 40px on desktop, 24px on mobile
Text content should not exceed 720px width for readability
Hero sections may use full container width but text stays constrained
Print materials must have minimum 20mm margins on all sides
Posters should use at least 10% margin on each side
Margin Do
  • Use consistent margins on all four sides
  • Give content room to breathe
  • Use larger margins on posters and presentations
  • Match margins to the spacing scale
  • Increase margins on larger screens
  • Keep margins symmetrical left and right
Margin Don't
  • Don't use zero margins on any layout
  • Don't let text touch the edges of the screen
  • Don't use different margins on left and right
  • Don't use arbitrary margin values
  • Don't reduce margins to fit more content
  • Don't ignore margins on mobile devices

Alignment Rules

Alignment is one of the most important layout principles. Proper alignment creates order, structure, and professionalism. Koleex uses primarily left alignment with center alignment reserved for specific use cases.

Left Alignment (Default)
  • Body text and paragraphs
  • Lists and bullet points
  • Form labels and inputs
  • Navigation items
  • Table content
  • Card content and descriptions
  • Document and report text
  • Sidebar content
  • Footer links and information
Center Alignment (Exception)
  • Hero section headlines
  • Page titles on landing pages
  • Short taglines and slogans
  • Call-to-action buttons
  • Single-line captions under images
  • Empty state messages
  • Modal dialog titles
  • Icon + label combinations
  • Poster headlines (when appropriate)

Never Use Right Alignment

Right alignment is not part of the Koleex design system (except in RTL languages like Arabic). Right-aligned text looks unprofessional and breaks the visual structure of the layout. Justified text is also not allowed — it creates uneven word spacing.

Layout for Documentation

Documentation layouts must feel editorial and technical at the same time. They should communicate precision, structure, and professionalism — like a technical manual from a premium engineering brand.

Documentation must look:

Clean and minimal — no visual clutter or unnecessary decoration
Structured with clear headings, subheadings, and body text
Well-spaced with generous margins and padding
Grid-aligned with consistent column widths
Professional like Apple, BMW, or IBM documentation
Easy to navigate with clear section boundaries
Consistent across all pages and sections
Focused on content readability above all else

Documentation must NOT look like:

A casual blog with inconsistent styling
A Word document with default formatting
A presentation with too many colors and effects
A marketing flyer with decorative elements
A crowded layout with no breathing room
An unstructured page without clear hierarchy

Layout Don't

These are the 23 layout mistakes that must never appear in any Koleex design. Violating these rules will result in rejected designs during brand review.

Don't place elements without grid alignment
Don't use arbitrary spacing values outside the defined scale
Don't use zero margins on any layout
Don't let content touch the edges of the screen or page
Don't crowd content — if in doubt, add more space
Don't mix alignment styles within the same content block
Don't use right alignment for body text (except RTL languages)
Don't use justified text alignment
Don't create layouts without clear visual hierarchy
Don't use inconsistent column widths within the same grid
Don't ignore responsive behavior — layouts must work on all devices
Don't use different grid systems across different brand materials
Don't stretch text content beyond 720px width
Don't use decorative layout elements (borders, shadows, gradients) for structure
Don't place logos or brand marks off-grid
Don't use random padding values inside content blocks
Don't create asymmetric layouts without intentional purpose
Don't reduce margins to fit more content on the page
Don't use more than 3 column variations on a single page
Don't skip the section structure pattern in documentation
Don't allow uneven vertical spacing between sections
Don't center-align long paragraphs of text
Don't publish any layout without grid verification
